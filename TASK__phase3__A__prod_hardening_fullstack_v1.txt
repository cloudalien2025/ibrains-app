STEP 0 — Collect environment facts (write results into docs/PROD_HARDENING_NOTES.md)

STEP 1 — Create a production systemd service for the Next.js app
- Ensure `npm ci` is used (or `npm install` only if lockfile is missing)
- Run `npm run build`
- Create a systemd unit:
  /etc/systemd/system/ibrains-app.service
  Requirements:
  - WorkingDirectory=/root/ibrains-app
  - EnvironmentFile=/root/ibrains-app/.env.production.local (create if missing)
  - ExecStart should run: `npm run start -- --port 3001`
  - Restart=always
  - User=root (acceptable for now) OR create a dedicated user `ibrains` if feasible without breaking permissions
  - StandardOutput=append:/var/log/ibrains-app/app.log
  - StandardError=append:/var/log/ibrains-app/app.log
- Create log directory /var/log/ibrains-app with correct perms
- Enable + start service
- Verify: `curl -sS http://127.0.0.1:3001/api/health`

STEP 2 — Nginx reverse proxy for the domain
- Install nginx if missing
- Create nginx site config:
  /etc/nginx/sites-available/ibrains-app.conf
  Must:
  - server_name <SET_DOMAIN>;
  - proxy_pass http://127.0.0.1:3001;
  - proper proxy headers (Host, X-Forwarded-For, X-Forwarded-Proto)
  - reasonable timeouts
  - gzip optional
  - deny access to hidden files
- Enable site via symlink into sites-enabled
- Disable default site if conflicting
- `nginx -t` then reload
- Validate on HTTP first:
  `curl -I http://<SET_DOMAIN>/api/health`

STEP 3 — HTTPS via Let’s Encrypt (certbot)
- Install certbot + nginx plugin
- Obtain certificate for <SET_DOMAIN> using certbot nginx method
- Force redirect HTTP -> HTTPS
- Verify:
  `curl -I https://<SET_DOMAIN>/api/health`
- Confirm auto-renew:
  `systemctl status certbot.timer`
  `certbot renew --dry-run`

STEP 4 — Firewall lockdown (UFW)
- Install ufw if missing
- Allow OpenSSH
- Allow Nginx Full (80/443)
- Deny everything else incoming by default
- Ensure port 3001 is NOT exposed publicly (only localhost)
- Verify UFW status and rules

STEP 5 — Logging + rotation
- Ensure nginx logs in /var/log/nginx
- Ensure app logs in /var/log/ibrains-app/app.log
- Add logrotate config:
  /etc/logrotate.d/ibrains-app
  Rotate daily, keep 14, compress, delaycompress, copytruncate
- Confirm logrotate runs without errors:
  `logrotate -d /etc/logrotate.d/ibrains-app`
- Confirm nginx logrotate already exists and is active (Ubuntu default)
